# 基于大模型的文本批量标注系统 — 开发需求文档

## 一、项目概述

基于 `async_batch_inference` 库，开发一个带 GUI 的文本批量标注系统。系统负责所有业务逻辑（方案管理、prompt 拼接、密钥存储、任务控制等），库仅作为纯推理黑盒被调用。最终将系统与库一起打包为免安装可执行程序。

### 核心原则

- **`async_batch_inference` 库零改动、零子类化**，仅通过其公开 API 调用
- 系统拼好 messages 后直接传给库，库只负责推理
- 密钥由系统加密存储，解密后以明文传给 `AsyncBatchConfig`
- 任务终止由系统控制批次实现，不修改库

## 二、功能需求

### 2.1 标注方案管理

系统需提供界面，支持标注方案的增删改查：

- **方案列表页**：卡片网格展示所有方案，显示名称和更新时间，支持新建、删除、点击进入编辑
- **方案编辑页**：可修改方案名称、prompt 头、标注类
- **标注类操作**：增删改标注类、调整顺序
- **标注类型**：
  - 类内可设置可选标注类型（如"正面/负面/中性"），模型从选项中选择
  - 类内也可不设置标注类型，由大模型根据类说明自行决定输出
- **JSON 导入导出**：方案可通过 JSON 文件导入或导出

#### 数据模型

```json
{
  "id": "scheme_001",
  "name": "方案名称",
  "prompt_header": "放在 system message 开头的指令文本",
  "classes": [
    {
      "id": "cls_001",
      "name": "类名",
      "description": "指导大模型如何标注该类的说明文字",
      "label_type": "select",
      "options": ["选项1", "选项2", "选项3"]
    },
    {
      "id": "cls_002",
      "name": "类名",
      "description": "该类的说明文字",
      "label_type": "free",
      "options": []
    }
  ],
  "created_at": "2026-05-19T10:00:00",
  "updated_at": "2026-05-19T10:00:00"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | str | 方案唯一标识，自动生成 |
| `name` | str | 方案名称，用户可编辑 |
| `prompt_header` | str | 放在 system message 开头的指令，定义大模型角色和任务背景 |
| `classes` | list | 标注类列表 |
| `classes[].id` | str | 类唯一标识，自动生成 |
| `classes[].name` | str | 标注类名称 |
| `classes[].description` | str | 标注类说明，告诉大模型该类如何标注 |
| `classes[].label_type` | str | `"select"` 表示从 options 中选，`"free"` 表示大模型自由输出 |
| `classes[].options` | list[str] | 可选标注类型列表，`label_type` 为 `"free"` 时为空列表 |
| `created_at` | str | ISO 8601 格式创建时间 |
| `updated_at` | str | ISO 8601 格式更新时间 |

### 2.2 文件标注执行

用户操作流程：

1. 上传 Excel 或 CSV 文件
2. 选择文件中的目标列
3. 选择标注方案
4. 选择使用的模型（从已配置的模型列表中选）
5. 执行标注

系统内部处理：

1. 用 pandas 读取文件，提取目标列的所有文本
2. 对每行文本，结合标注方案拼接完整的 messages（见 2.3）
3. 将 messages 按小批次拆分，逐批调用 `AsyncBatchInference.batch()`
4. 每批完成后将标注结果经过三层校验后回写 DataFrame：
   - **第一层 prompt 约束**：在 system prompt 中强制 JSON 格式输出和选项取值范围
   - **第二层 answer_processor 解析**：JSON 提取（含 markdown 代码块兼容）、类型校验
   - **第三层 select 校验**：精确匹配 → 模糊匹配（含编辑距离），无法匹配标记 `[?]` 前缀
5. 全部完成后，用户可下载标注后的文件

### 2.3 Prompt 拼接规则

系统将标注方案与用户文本拼接为标准 messages 格式，直接传给库：

```
messages = [
  {
    "role": "system",
    "content": "<prompt_header>\n\n<各标注类说明及格式指令>"
  },
  {
    "role": "user",
    "content": "<目标列的行文本>"
  }
]
```

system message 拼接逻辑：

1. 以 `prompt_header` 开头
2. 逐个列出标注类的名称和说明
3. 若该类 `label_type` == `"select"` 且 `options` 非空，列出可选值，并加约束"你必须且只能从以上可选值中选择一个"
4. 若该类 `label_type` 为 `"free"`，指明由模型根据描述自行判断
5. 末尾添加输出格式指令：必须输出 JSON 对象、给出示例格式、对于有可选值字段值必须来自可选值

### 2.4 密钥安全存储

- 使用 `cryptography` 库的 Fernet 对称加密
- 将 `api_key`、`base_url`、`models`（模型列表）序列化后加密，写入二进制文件 `data/secret.dat`
- 支持旧版本迁移：单项 `model` 字段自动转为 `models` 列表
- 系统启动时解密读取明文，直接传入 `AsyncBatchConfig`
- UI 中配置页支持增删模型、逐模型测试连接
- Fernet 加密密钥可由环境变量 `TAS_FERNET_KEY` 指定，否则由设备 COMPUTERNAME/HOSTNAME 派生
- 仅暴露加密后的 secret.dat 二进制文件，UI 中不显示密钥明文

调用方式：

```python
api_key, base_url, models = security.decrypt_config(data_dir)
config = AsyncBatchConfig(api_key=api_key, base_url=base_url, model=models[0])
engine = AsyncBatchInference(config)
```

### 2.5 任务管理与监控

#### 小批次 + 自控终止策略

将总数据拆成小批次（如每批 20 条），逐批调用库的 `batch()` 方法：

```
总数据 1000 行，批次大小 20 → 50 个批次

批次1: engine.batch(prompts[0:20])    → 完成，进度 20/1000
批次2: engine.batch(prompts[20:40])   → 完成，进度 40/1000
批次3: engine.batch(prompts[40:60])   → 执行中...
用户点击"终止" → 不再发送后续批次，批次3结果丢弃，任务标记为 terminated
已完成批次1-2的结果仍可下载
```

#### 任务记录

每个任务需记录以下信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | str | 任务唯一标识 |
| `file_name` | str | 上传的文件名 |
| `scheme_id` | str | 使用的标注方案 ID |
| `scheme_name` | str | 标注方案名称（快照） |
| `model` | str | 使用的模型名称 |
| `status` | str | `running` / `completed` / `terminated` |
| `total_rows` | int | 总行数 |
| `completed_rows` | int | 已完成行数 |
| `created_at` | str | ISO 8601 创建时间 |
| `finished_at` | str | ISO 8601 结束时间 |
| `duration` | str | 任务用时（如 "2m 30s"、"1h 15m"），前端显示为友好格式 |
| `output_file` | str | 输出文件路径 |

#### 任务操作

- 查看当前运行中及历史任务列表（含用时、创建时间、结束时间）
- 查看任务进度（已完成/总数，进度条）
- WebSocket 实时推送运行中任务进度
- 下载已完成任务的标注结果文件
- 终止运行中的任务
- 删除非运行中的历史任务（二次确认）

## 三、数据持久化

### 存储方式：JSON 文件夹

不使用数据库，所有持久化数据以 JSON 文件存储在 `data/` 目录下，方便手动编辑和复制。

### 目录结构

```
data/
├── schemes/          # 标注方案，每个方案一个 JSON 文件
│   ├── scheme_001.json
│   ├── scheme_002.json
│   └── ...
├── tasks/            # 任务记录，每个任务一个 JSON 文件
│   ├── task_001.json
│   └── ...
├── uploads/          # 用户上传的 Excel/CSV 文件
│   └── ...
├── outputs/          # 标注完成后的输出文件（每个标注类单独一列）
│   └── ...
└── secret.dat        # 加密存储的密钥二进制文件（含 api_key, base_url, models）
```

### 文件命名规则

- 方案文件：`scheme_{id}.json`，id 使用 UUID 或递增编号
- 任务文件：`task_{id}.json`

### 注意事项

- 系统为单用户桌面应用，无并发写入冲突
- 读写 JSON 文件时需处理文件不存在的情况（首次启动自动创建目录）
- 方案的 JSON 导入导出格式与存储格式完全一致，零转换
- 密钥格式支持新旧版本兼容迁移

## 四、技术选型

| 层次 | 技术 | 理由 |
|------|------|------|
| 后端框架 | FastAPI | 原生 async，与库的 asyncio 引擎契合；WebSocket 原生支持 |
| 前端 | Vue 3 + Element Plus | 表格/表单/对话框/进度条组件丰富 |
| 数据持久化 | JSON 文件 | 手动可编辑、可直接复制大模型生成的内容 |
| 文件处理 | pandas + openpyxl | 读写 Excel / CSV |
| 密钥加密 | cryptography (Fernet) | 对称加密写二进制文件，密钥可由环境变量或机器特征派生 |
| 实时通信 | WebSocket (FastAPI) | 推送批次完成进度 |
| 打包 | PyInstaller (onedir) | 后端 + 前端静态文件 + async_batch_inference 打包为免安装目录 |
| 前端构建 | Vite | 快速 HMR 开发、高效生产构建 |
| 全局样式 | CSS 自定义属性 | 无依赖的 Element Plus 主题定制，保持项目轻量 |

## 八、调用库的方式（完整示例）

```python
from async_batch_inference import AsyncBatchConfig, AsyncBatchInference
from core.security import decrypt_config
from core.prompt_builder import build_messages
from core.scheme_manager import SchemeManager

# 1. 读取密钥 → 明文传入库
api_key, base_url, models = decrypt_config(data_dir)
config = AsyncBatchConfig(api_key=api_key, base_url=base_url, model=task.model or models[0])
engine = AsyncBatchInference(config)

# 2. 加载方案
manager = SchemeManager("data/schemes")
scheme = manager.get("scheme_001")

# 3. 拼接 messages
prompts = [build_messages(scheme, row_text) for row_text in text_list]

# 4. 小批次逐批调用
BATCH_SIZE = 20
for i in range(0, len(prompts), BATCH_SIZE):
    if task.cancelled:
        break
    batch = prompts[i:i+BATCH_SIZE]
    results = await engine.batch(batch)
    # 5. 三层校验
    processed = [process_answer(r.get("answer", ""), scheme) for r in results]
    save_results(processed)
    notify_progress(i + len(batch), len(prompts))
```

## 九、打包方案

### 打包模式

采用 PyInstaller `onedir` 模式（非 `onefile`）：
- 产物为一个文件夹，包含 exe 和 `_internal/` 依赖目录
- 启动无需解压，比单文件模式更快更稳定
- 最终压缩为 zip 分发，用户解压即可运行

### 分发包目录结构

```
TextAnnotationSystem/
├── TextAnnotationSystem.exe        # 可执行文件
├── _internal/                      # Python 解释器 + 所有依赖 + 前端静态资源
│   ├── async_batch_inference/      # 库文件
│   ├── static/                     # 前端构建产物
│   └── ...（Python 环境）
├── data/                           # 用户数据目录（首次运行自动创建子目录）
│   ├── schemes/
│   ├── tasks/
│   ├── uploads/
│   ├── outputs/
│   └── secret.dat
└── data_dir.txt                    # 数据存储路径配置（默认 "data"）
```

### 存储路径配置

用户可通过以下方式自定义数据存储位置（优先级从高到低）：

1. **环境变量 `TAS_DATA_DIR`**：指定绝对路径
2. **`data_dir.txt` 文件**：exe 同级，写入相对或绝对路径
3. **默认路径**：exe 同级的 `data/` 目录

### 打包命令

通过构建脚本自动化执行：

```powershell
powershell -ExecutionPolicy Bypass -File "scripts/build_windows.ps1"
```

脚本自动完成：停止旧进程 → 构建前端 → 安装依赖 → PyInstaller 打包 → 生成 data_dir.txt → 压缩 zip

### 其他配置

- `TAS_PORT` 环境变量可修改服务端口（默认 8765）
- `TAS_FERNET_KEY` 环境变量可指定加密密钥

### 分发说明

- 将 `dist/TextAnnotationSystem-windows.zip` 发给 Windows 用户
- 用户解压后双击 `TextAnnotationSystem.exe` 即可使用
- 无需安装 Python、Node.js 或任何依赖
- 首次运行可能被 Windows Defender 提示（未签名的 exe），选择"仍要运行"即可

## 十、开发约束

1. **不动 `async_batch_inference` 库**：所有功能在系统内部实现，库仅通过公开 API 调用
2. **JSON 文件持久化**：所有方案和任务数据以 JSON 文件存储，格式支持手动编辑
3. **密钥不明文暴露**：api_key 必须加密存储在二进制文件中，UI 不展示明文
4. **小批次控制**：通过小批次逐步提交实现进度追踪和终止能力
5. **单用户桌面场景**：无需考虑并发冲突和多用户权限
6. **Python 后端 + Vue 前端**：FastAPI 提供REST API 和 WebSocket，Vue 3 构建静态前端
7. **代码无中文注释**：代码注释使用英文
8. **源码模式下自动查找 async_batch_inference**：从 main.py 向上逐级父目录搜索
9. **打包模式下资源自包含**：exe 通过 `sys._MEIPASS` 读取包内资源，不依赖外部路径
10. **uvicorn 打包兼容**：frozen 模式下禁用默认日志配置，避免 GUI exe 中 `NoneType.isatty` 报错
